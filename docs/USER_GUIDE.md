## 1. ภาพรวมระบบ

SmartMA คือระบบบริหาร Ticket และสัญญา Maintenance สำหรับบริษัท IT ช่วยให้ทีมงานติดตามงานซ่อมบำรุงฮาร์ดแวร์และซอฟต์แวร์ได้ในที่เดียว พร้อม Sync สถานะอัตโนมัติกับ Jira

**ผู้ใช้งานมี 3 ระดับ:**

| Role | หน้าที่ |
|------|--------|
| **Admin (ผู้ดูแลระบบ)** | เข้าถึงได้ทุกฟีเจอร์ จัดการผู้ใช้และสิทธิ์ |
| **PM (Project Manager)** | บริหารสัญญา เปิด/แก้ไข Ticket ดูรายงาน |
| **Staff** | ดู Ticket ที่ได้รับมอบหมาย และอัปเดตสถานะ |

## 2. การเข้าสู่ระบบ

1. เปิดเบราว์เซอร์ไปที่ URL ของระบบ
2. กรอก **อีเมล** และ **รหัสผ่าน**
3. กด **เข้าสู่ระบบ**

ระบบจะพาไปหน้า **สัญญา / โปรเจกต์** โดยอัตโนมัติ

> หากกรอกข้อมูลไม่ถูกต้อง จะแสดงข้อความ "อีเมลหรือรหัสผ่านไม่ถูกต้อง"

## 4. สัญญา / โปรเจกต์

### 4.1 ดูรายการสัญญา

หน้านี้แสดงภาพรวมสัญญาทั้งหมดที่คุณดูแล ประกอบด้วย:

**การ์ดสรุป (Stats) ด้านบน:**
- **สัญญาทั้งหมด** — จำนวนสัญญาที่มีในระบบ
- **Tickets เปิดอยู่** — Ticket ที่ยัง Open ทั้งหมด
- **In Progress** — Ticket ที่กำลังดำเนินการ
- **Jira Last Sync** — สถานะการ Sync ล่าสุด

**แถบกรองสัญญา:**
- **ทั้งหมด** — แสดงทุกสัญญา
- **ภาครัฐ** — กรองเฉพาะสัญญาหน่วยงานรัฐ
- **เอกชน** — กรองเฉพาะสัญญาบริษัทเอกชน

**ช่องค้นหา:** พิมพ์ชื่อสัญญาเพื่อกรองทันที

### 4.2 การ์ดสัญญา

แต่ละสัญญาแสดงข้อมูล:
- ชื่อสัญญาและ Badge ประเภท (ภาครัฐ / เอกชน)
- วันเริ่ม — วันสิ้นสุดสัญญา และประเภทงาน (Software / Hardware)
- จำนวน Ticket แยกตาม Open / In Progress / Done
- สถานะการเชื่อมต่อ Jira

**กดที่การ์ด** เพื่อเข้าดู Ticket ในสัญญานั้น

### 4.3 สร้างสัญญาใหม่

1. กดปุ่ม **+ เพิ่มสัญญาใหม่**
2. กรอกข้อมูล:
   - **ชื่อสัญญา** (จำเป็น)
   - **ประเภท** — ภาครัฐ หรือ เอกชน
   - **ประเภทงาน** — Software / Hardware / ทั้งคู่
   - **วันเริ่มต้น** และ **วันสิ้นสุด**
3. กด **บันทึก**

> **สิทธิ์:** Admin และ PM สามารถสร้างสัญญาได้

## 5. Ticket

### 5.1 ดูรายการ Ticket

เข้าจากการ์ดสัญญา หรือเมนู **Tickets** ใน Sidebar

หน้าแสดง:
- **Header สัญญา** — ชื่อ, ประเภท, สรุป Open / In Progress / Done
- **แถบกรองสถานะ** — ทั้งหมด / Open / In Progress / Done
- **ช่องค้นหา** — ค้นหาจากหัวข้อ Ticket
- **ตาราง Ticket** — แสดง #, หัวข้อ, ประเภท, Priority, Status, ผู้รับผิดชอบ

### 5.2 เปิด Ticket ใหม่

1. กดปุ่ม **+ เปิด Ticket ใหม่**
2. กรอกข้อมูล:
   - **หัวข้อ** (จำเป็น)
   - **ประเภท** — Hardware (HW) หรือ Software (SW)
   - **Priority** — เลือกระดับความเร่งด่วน (ดูตารางด้านล่าง)
   - **ผู้รับผิดชอบ** — เลือก Staff ที่ต้องการมอบหมาย
   - **Jira Key** — ถ้า Ticket นี้มีใน Jira ให้ระบุ Key เช่น `PROJ-123`
   - **รายละเอียด** — อธิบายปัญหาเพิ่มเติม
3. กด **บันทึก**

**ระดับ Priority:**

| Priority | ความหมาย |
|----------|----------|
| P1 — ด่วนมาก | ระบบล่ม / กระทบการทำงานทันที |
| P2 — ด่วน | ส่งผลกระทบสูง ต้องแก้ไขเร็ว |
| P3 — ปกติ | ปัญหาทั่วไป ไม่กระทบการทำงานหลัก |
| P4 — ต่ำ | ปรับปรุงเล็กน้อย / ไม่เร่งด่วน |

### 5.3 รายละเอียด Ticket

กดที่ Ticket ในตารางเพื่อเปิดหน้ารายละเอียด ประกอบด้วย:

**ฝั่งซ้าย:**
- **หัวข้อ Ticket** และ Status Flow แสดงขั้นตอนปัจจุบัน
- **รายละเอียด** — คำอธิบายปัญหา
- **ความคิดเห็น** — พิมพ์ข้อความแล้วกด **ส่ง** เพื่อเพิ่ม comment

**ฝั่งขวา:**
- Status, Priority, Type, Assignee, วันที่อัปเดต
- ปุ่ม **เปลี่ยน Status** — เลื่อนสถานะไปขั้นถัดไป
- Panel **Jira** — แสดง Jira Key และลิงก์ (ถ้ามี)

### 5.4 Status Flow

Ticket ไหลตามลำดับนี้เท่านั้น:

```
OPEN  →  IN PROGRESS  →  RESOLVED  →  CLOSED
```

ปุ่มเปลี่ยน Status ตาม Flow:

| Status ปัจจุบัน | ปุ่มที่แสดง |
|----------------|------------|
| OPEN | รับงาน (→ In Progress) |
| IN PROGRESS | แก้ไขแล้ว (→ Resolved) |
| RESOLVED | ปิด Ticket (→ Closed) |
| CLOSED | — (ไม่มีปุ่ม) |

## 6. รายงาน

เมนู **รายงาน** แสดงภาพรวมสถิติทั้งระบบ

### การ์ดสรุป

- **Tickets ทั้งหมด** — จำนวน Ticket ทั้งหมดในระบบ
- **Open** — กำลังรอดำเนินการ
- **In Progress** — กำลังดำเนินการ
- **ปิดแล้ว** — Resolved + Closed รวมกัน

### กราฟ Tickets ตามสัญญา

Bar chart แสดงแต่ละสัญญาแยกตามสี:
- 🔴 **Open**
- 🟠 **In Progress**
- 🟢 **Done (Resolved + Closed)**

### ตารางสรุปตามสัญญา

แสดงทุกสัญญาพร้อมจำนวน Ticket แต่ละสถานะ และยอดรวม

> **สิทธิ์:** Admin และ PM เท่านั้น

## 7. Jira Sync

### 7.1 ภาพรวม

หน้า **Jira Sync** ใช้สำหรับดึงข้อมูลสถานะล่าสุดจาก Jira มาอัปเดต Ticket ในระบบ

- **Sync Status** — แสดง Connected (มี Jira Config อย่างน้อย 1 สัญญา)
- **Last Sync** — เวลาที่ Sync ล่าสุด
- **Sync Log** — บันทึกผลการ Sync แต่ละครั้ง

### 7.2 Manual Sync

กดปุ่ม **Sync Now** เพื่อ Sync ทันที

ระบบจะ:
1. ดึง Ticket ทั้งหมดที่มี Jira Key
2. เรียก Jira API เพื่อดูสถานะล่าสุด
3. อัปเดต Status ใน SmartMA ตาม Jira
4. บันทึกผลลง Sync Log

### 7.3 ตั้งค่า Jira ต่อสัญญา

การเชื่อมต่อ Jira ตั้งค่าต่อสัญญาผ่านหน้า **Settings → Jira API** โดยใส่:
- **Jira Host** — เช่น `company.atlassian.net`
- **Email** — อีเมล Jira account
- **API Token** — สร้างที่ https://id.atlassian.com → Security → API tokens
- **Project Key** — เช่น `PROJ`

## 8. จัดการผู้ใช้งาน (Admin)

> เมนูนี้มองเห็นเฉพาะ **Admin** เท่านั้น

### 8.1 ดูรายชื่อผู้ใช้งาน

แสดงตารางผู้ใช้ทั้งหมดพร้อม:
- ชื่อ-นามสกุล และ Avatar
- อีเมล
- Role (ผู้ดูแลระบบ / Project Manager / Staff)
- สถานะ (Active / Inactive)

### 8.2 เพิ่มผู้ใช้ใหม่

1. กดปุ่ม **+ เพิ่มผู้ใช้**
2. กรอก ชื่อ, อีเมล, รหัสผ่าน และ Role
3. กด **บันทึก**

### 8.3 แก้ไข Role หรือปิดการใช้งาน

กดที่ผู้ใช้ในตาราง แล้วเลือก:
- **เปลี่ยน Role** — Admin / PM / Staff
- **ปิดการใช้งาน** — ผู้ใช้จะ Login ไม่ได้ แต่ข้อมูลยังอยู่ในระบบ

## 9. ตั้งค่าระบบ

เมนู **ตั้งค่า** มี 4 แท็บ:

### ทั่วไป
- **ชื่อบริษัท** — ชื่อที่แสดงในระบบ
- **ภาษาเริ่มต้น** — ไทย / อังกฤษ
- **Timezone** — โซนเวลา (ค่าเริ่มต้น: Asia/Bangkok UTC+7)

### การแจ้งเตือน
- เปิด/ปิดการแจ้งเตือนสำหรับเหตุการณ์ต่าง ๆ เช่น Ticket ใหม่, สถานะเปลี่ยน

### Jira API
- ตั้งค่า Jira credentials ต่อสัญญา (ดูรายละเอียดที่ [หัวข้อ 7.3](#73-ตั้งค่า-jira-ต่อสัญญา))

### Danger Zone
- ตัวเลือกที่มีผลกับระบบโดยรวม ควรระวังในการใช้งาน
